SPORTPILOT 0.18.0

PRÉPARATION

1. Vérifier que la branche feature/account-data-spaces-0.18.0 est propre.
2. Arrêter le serveur de développement avec Ctrl + C.
3. Exporter une sauvegarde JSON v7 récente et la conserver hors de l’application.
4. Ne supprimer ni IndexedDB, ni les données Safari, ni la PWA.
5. Vérifier que .env.local, dexie-cloud.json et dexie-cloud.key restent ignorés par Git.
6. Vérifier les origines Dexie Cloud :

   npx dexie-cloud whitelist

7. Utiliser une origine autorisée, notamment :

   http://localhost:5173
   http://127.0.0.1:5173
   https://sport-nutrition-tracker.axel-cottrant.workers.dev

VALIDATION LOCALE

8. Installer les dépendances :

   npm ci

9. Lancer les contrôles :

   npm run release:verify
   npm run test:e2e

10. Vérifier que Paramètres affiche la version 0.18.0.
11. Vérifier que les données historiques restent dans « Espace local invité ».
12. Vérifier qu’un compte connecté ne charge jamais les données d’un autre compte.
13. Vérifier la page « Compte et appareils ».
14. Vérifier le fonctionnement hors connexion de l’espace actif.

PUBLICATION

15. Committer la préparation 0.18.0 sur feature/account-data-spaces-0.18.0.
16. Fusionner cette branche dans develop avec un commit de fusion explicite.
17. Après validation de develop, fusionner develop dans main.
18. Créer le tag annoté v0.18.0 sur le commit publié dans main.
19. Pousser main, develop et le tag v0.18.0.
20. Le push de main déclenche le déploiement Cloudflare.

VALIDATION APRÈS DÉPLOIEMENT

21. Vérifier la version 0.18.0 sur ordinateur et iPhone 15.
22. Vérifier la conservation des espaces invité et compte.
23. Tester un changement compte A / compte B sans fuite de données.
24. Tester la déconnexion, la désassociation et la réassociation.
25. Créer puis synchroniser une pesée entre deux appareils du même compte.
26. Vérifier le mode hors connexion puis la reprise du réseau.

VERSIONS DE DONNÉES

Schéma Dexie principal : version 8.
Sauvegarde JSON : version 7.
Registre local des espaces : version 1.

Aucune migration de table n’est requise pour 0.18.0. Les espaces de comptes sont créés dans des bases IndexedDB distinctes. La base sportpilot-local-database demeure l’espace invité historique.
