SPORTPILOT 0.19.0

PRÉPARATION

1. Vérifier que la branche feature/sports-sync-0.19.0 est propre.
2. Arrêter le serveur de développement avec Ctrl + C.
3. Fermer tous les onglets localhost:5173 et 127.0.0.1:5173.
4. Exporter une sauvegarde JSON v7 récente et la conserver hors de l’application.
5. Ne supprimer ni IndexedDB, ni les données Safari, ni la PWA.
6. Vérifier que .env.local, dexie-cloud.json et dexie-cloud.key restent ignorés par Git.
7. Vérifier les origines Dexie Cloud avec npx dexie-cloud whitelist.

VALIDATION LOCALE

8. Installer les dépendances avec npm ci.
9. Lancer npm run release:verify.
10. Lancer npm run test:e2e.
11. Vérifier que Paramètres affiche la version 0.19.0.
12. Vérifier que le compte connecté ouvre son espace isolé.
13. Vérifier les synchronisations des pesées, activités, objectifs et données de musculation.
14. Après chaque synchronisation, relancer l’analyse et confirmer 0 différence.
15. Vérifier une séance complète sur deux navigateurs : séance, exercices et séries.
16. Vérifier les suppressions d’une série et d’un exercice de séance.
17. Vérifier le fonctionnement hors connexion de l’espace actif.

PUBLICATION

18. Committer la préparation 0.19.0 sur feature/sports-sync-0.19.0.
19. Fusionner cette branche dans develop avec un commit de fusion explicite.
20. Après validation de develop, fusionner develop dans main.
21. Créer le tag annoté v0.19.0 sur le commit publié dans main.
22. Pousser main, develop et le tag v0.19.0.
23. Le push de main déclenche le déploiement Cloudflare.

VALIDATION APRÈS DÉPLOIEMENT

24. Vérifier la version 0.19.0 sur ordinateur et iPhone 15 sous iOS 26.
25. Vérifier la connexion au compte et l’ouverture de l’espace existant.
26. Synchroniser chaque domaine sur ordinateur puis sur iPhone.
27. Vérifier l’absence de doublons, de séance partielle et de série orpheline.
28. Tester le mode hors connexion puis la reprise du réseau.
29. Tester un changement compte A / compte B sans fuite de données.

VERSIONS DE DONNÉES

Base Dexie Cloud : version 5.
Runtime local cloud : sportpilot-sync-runtime-0.19.0-v5.
Schéma Dexie principal : version 8.
Sauvegarde JSON : version 7.
Registre local des espaces : version 1.

Aucune migration de la base métier principale n’est requise pour 0.19.0.
